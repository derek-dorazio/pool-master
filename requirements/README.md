# Requirements

The `requirements/` folder is the product-definition area for the repo.

## Structure

- `requirements/reference/`
  - seed materials for discovery
  - screenshots, rough docs, SME notes, sketches, or competitor references
- `requirements/product-overview/`
  - Piper's high-level product-discovery output
- `requirements/product-requirements/`
  - Pam's refined product-requirement output

## Execution Authority

These requirement artifacts are design inputs and handoff materials.

Active execution tracking still belongs in `plans/`.
